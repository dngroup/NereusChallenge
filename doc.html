How to use the MS-Stream demonstration

In this demonstration, you can:

  1. Add or remove servers by clicking on their icons;
  2. Consider simultaneously using more or less servers by using the blue sliders below the video frame;
  3. Modify servers' upload throughput by using the green sliders on the bottom right side of the webpage;
  4. Click on "Infos" in the top right of the video to get metrics about the bitrate of the video, the number of servers used and the buffer of the video;
  5. Switch back to a DASH use-case by using one server only.

Tips for the demo

  • Step 1 - At the beginning, the player is using only one server, following the DASH standard. As the bandwidth of the server is limited, the quality of the video is 2 Mbps.
  • Step 2 - Increase the number of simultaneous servers to 3. It will automatically add servers. Now, the player is using 3 servers and can aggregate their bandwidth to play the 6 Mbps quality.
  • Step 3 - Decrease the bandwidth of one of the servers. The global bandwidth of the streaming session decreases. The contribution of the modified server is less than the 2 others. The video quality decreases, but only a little.
  • Step 4 - Add a 4th server by clicking on it on the schema. The player switches the worst server with the new one and since the new server has a better bandwidth, the player keeps this new server.
